기강 · Claude Code 확장하기

능력을 더하는 법 — 플러그인 & MCP

기본 Claude도 강하지만, 플러그인(확장팩)과 MCP(외부 연결 규격)로 능력을 더 붙일 수 있습니다. 둘 다 누구나 만들어 GitHub에 공유하니, 찾아서 가져다 쓰면 됩니다.

한눈에

“능력을 더한다”는 두 갈래입니다. 하나는 Claude Code 자체에 기능 묶음을 설치하는 것(플러그인), 다른 하나는 Claude를 외부 서비스에 연결하는 것(MCP)입니다. 마지막에 아무거나 가져다 쓸 때의 보안 주의까지 짚습니다.

① 플러그인Claude Code에 끼우는 “확장팩” (스킬·명령 묶음)
② MCPClaude를 외부 서비스에 잇는 “표준 어댑터”
③ GitHub에서 찾아쓰기카카오톡 MCP 등 누구나 올린 것을 검색
④ 보안 주의가져온 것은 내 권한으로 돈다 — 신뢰부터
⑤ 비용·메모리많이 깔수록 토큰을 먹는다 — 자주 쓰는 것만
1

플러그인 — Claude Code에 끼우는 “확장팩”

스킬·명령·에이전트를 통째로 추가하는 묶음. 마켓플레이스에서 설치합니다
확장팩 (Plugin)
한 줄 비유
브라우저 확장프로그램이나 게임 DLC처럼, 기본 Claude Code에 기능 한 묶음을 통째로 더하는 것.
무엇인가
  • 플러그인 하나엔 여러 스킬·슬래시 명령·에이전트·자동화(훅)가 들어 있습니다.
  • Claude Code의 마켓플레이스에서 골라 설치합니다. (이 교육에서 쓰는 일부 기능도 플러그인 출신입니다.)
대표 플러그인
Superpowers가장 유명
개발·작업 방법론을 가르치는 스킬 묶음 — 만들기 전 설계(brainstorming), 체계적 디버깅, 단계별 계획 등. (부록 2의 brainstorming이 바로 여기 출신) 설치 수십만 건.
github.com/obra/superpowers
Context7실무 인기
라이브러리·도구의 최신 공식 문서를 그때그때 가져와 Claude에게 먹입니다. AI 지식이 오래돼 생기는 “옛날 방식대로 알려주는” 문제를 줄여줍니다.
github.com/upstash/context7

이 밖에 Anthropic 공식 플러그인(frontend-design, feature-dev, code-review 등)도 있습니다. 모아둔 목록: github.com/Chat2AnyLLM/awesome-claude-plugins

핵심

플러그인 = Claude Code 확장팩. 백지에서 다 만들지 말고, 이미 잘 만든 기능 묶음을 설치해 쓴다.

2

MCP — Claude를 외부 서비스에 잇는 “표준 어댑터”

Model Context Protocol. AI와 외부 도구·데이터를 연결하는 공개 표준입니다
외부 연결 (MCP)
한 줄 비유
AI에 꽂는 USB 포트. 규격(표준)이 정해져 있어서, 한 번 맞춰 만든 장치(서버)를 여러 AI가 그대로 꽂아 씁니다.
무엇인가
  • MCP 서버는 특정 능력 하나를 제공하는 작은 프로그램입니다 — 예: 메일 읽기, 데이터베이스 조회, 캘린더 등록, 카카오톡 메시지 보내기.
  • 이 능력을 Claude에 연결하면, Claude가 그 서비스를 직접 다룰 수 있게 됩니다.
  • 핵심은 “누구나 만들어 공개”한다는 점 — GitHub에 수많은 MCP 서버가 올라와 있고, 검색해서 가져다 연결하면 됩니다.
예시 — 카카오톡 MCP는 실제로 있습니다 (모두 비공식·커뮤니티)
kakao-bot-mcp-server비공식
카카오 개발자 API로 “나에게 메시지 보내기” 등을 AI 에이전트와 연결. (카카오 공식 아님)
github.com/inspirit941/kakao-bot-mcp-server
kakao-api-mcp-server비공식
카카오맵·다음 검색 API를 AI가 쓰도록 연결 — 지도·검색 기능.
github.com/jeong-sik/kakao-api-mcp-server
kakaocli비공식 · UI 자동화
공식 API로 안 되는 일까지 — DB 읽기 + 화면 자동조작으로 카톡 메시지를 읽고 보냄. 접근 방식상 보안 주의가 가장 큰 유형(아래 4번 참고).
github.com/silver-flight-group/kakaocli
핵심

MCP = AI에 능력을 꽂는 표준 포트. 필요한 기능은 대개 GitHub에 누군가 이미 만들어 뒀다 — 검색해서 연결.

3

플러그인 vs MCP — 헷갈리지 않게

겹쳐 보이지만 역할이 다릅니다 (플러그인이 MCP를 품기도 합니다)

플러그인 (확장팩)

  • 어디에: Claude Code 안에 설치
  • 무엇을: 스킬·명령·에이전트·자동화 묶음
  • 목적: Claude의 일하는 방식을 늘림
  • 예: Superpowers, Context7

MCP (어댑터)

  • 어디에: 외부 서비스에 연결
  • 무엇을: 메일·DB·카톡 등 바깥 능력
  • 목적: Claude가 다룰 수 있는 대상을 늘림
  • 예: 카카오톡 MCP, 구글 캘린더 MCP

관계: 둘은 배타적이지 않습니다. 플러그인이 내부에 MCP 서버를 품고 배포되기도 합니다(예: Context7). 한 줄로: 플러그인은 “기능 묶음”, MCP는 “외부 연결 규격”.

핵심

플러그인은 Claude의 능력을, MCP는 Claude가 닿는 바깥세상을 넓힌다.

★

【주의】 GitHub에서 가져온 것은 “내 권한”으로 돈다

아무거나 연결하기 전에 — 신뢰·검증부터
보안 (Security)
왜 조심해야 하나
  • MCP 서버·플러그인은 내 컴퓨터에서 내 권한으로 실행됩니다 — 내 파일, 내 계정, 내 데이터에 접근할 수 있습니다.
  • 대부분 비공식·개인 제작입니다(카카오톡 MCP들도 전부 비공식). 악의가 없어도 버그·과도한 권한·방치된 코드일 수 있습니다.
  • 특히 로그인·UI 자동화·DB 직접 읽기 방식(예: kakaocli류)은 부록 1에서 말한 “자동 로그인 = 보안 솔루션에 탈취로 탐지” 문제와 직결됩니다. 회사 PC에서는 보안 정책 위반이 될 수 있습니다.
가져다 쓸 때 체크리스트

✕ 이러면 위험

  • 출처·코드 안 보고 바로 연결
  • 비번·토큰을 설정 파일에 그대로
  • 회사 PC에 비공식 메신저 자동화 설치

✓ 이렇게

  • 스타 수·최근 커밋·코드 한 번 확인
  • 권한은 꼭 필요한 범위만 부여
  • 민감 작업은 회사 보안 정책 먼저 확인
핵심

“GitHub에 있으니 안전”은 틀린 생각. 연결하는 순간 내 권한을 빌려주는 것 — 신뢰할 수 있는 것만, 권한은 최소로.

₩

【비용】 많이 깔수록 “토큰(메모리)”을 더 먹는다

설치만 해두고 안 써도, 켜져 있는 것만으로 토큰을 잡아먹습니다
비용·메모리 (Cost)
왜 그런가
  • 설치한 플러그인·MCP 서버는 대화를 시작할 때마다 각자의 도구 설명·명령 목록을 컨텍스트(AI의 메모리)에 미리 올립니다.
  • 그래서 많이 깔아둘수록 일을 시작하기도 전에 토큰을 차지합니다 — 안 쓰고 켜두기만 해도요.
무엇이 나빠지나
  • ① 비용 ↑ — 매 요청마다 더 많은 토큰을 깔고 시작합니다.
  • ② 여유 공간 ↓ — 실제 작업·대화에 쓸 메모리가 줄어듭니다.
  • ③ 정확도·속도 ↓ — 선택지(도구)가 너무 많으면 AI가 헷갈리거나 느려질 수 있습니다.
권장
  • “혹시 몰라서” 잔뜩 켜두지 말 것. 자주 쓰는 것만 남기고 나머지는 끄거나 제거하세요.
  • 필요할 때만 연결하는 습관 — 부록 2의 “빈도가 곧 비용” 원칙과 같은 맥락입니다.
핵심

플러그인·MCP는 켜져 있는 것만으로 토큰을 먹는다. 많이 깔지 말고 자주 쓰는 것만 남겨라.
