# 주의사항 & 보안

> AI를 잘못 쓰면 보안 사고가 날 수 있습니다. 올바른 사용법을 설명합니다.

## 로그인이 필요한 사이트 — 주의

AI에게 "이 사이트에 로그인해서 데이터 가져와"라고 시키는 것은
**보안 위험**이 될 수 있습니다.

이유:
- 계정 정보를 AI에게 직접 노출하게 됨
- 사이트 이용약관 위반일 수 있음
- 개인 계정 보안 위험

## 올바른 접근법

### 단발성 크롤링이 필요할 때: Playwright MCP

**비유:** "내가 문 열어줄 테니 그 다음은 네가 해"

1. Playwright MCP 플러그인 설치 (강사가 사전 설치 — 또는 [부록 E](../appendix/E-mcp.md) 참고)
2. Claude Code에게 지시:
```
브라우저 열어줘. 내가 로그인하면 그 다음에 데이터 가져와줘.
```
3. 브라우저가 열리면 내가 직접 로그인
4. 로그인 완료 후 AI가 이어서 작업

### 반복 자동화가 필요할 때: Selenium

Selenium은 웹브라우저를 자동으로 조작하는 도구다. Claude Code가 설치해준다.

**비유:** 내가 문 여는 열쇠를 직접 쓰되, 나머지는 자동화

```
매일 이 사이트에서 데이터를 가져오는 자동화를 만들어줘.
단, 내가 직접 로그인하는 단계는 내가 할 수 있게 만들어줘.
```

AI가 스크립트를 만들고, 로그인 부분만 사람이 직접 처리.

## 개인정보 & 민감 데이터 주의사항

AI에 **입력하면 안 되는 것:**
- 개인정보 (이름, 연락처, 주소)
- 내부 기밀 자료
- 비밀번호, API 키, 인증 토큰
- 아직 공개되지 않은 계획이나 전략

**괜찮은 것:**
- 익명화된 데이터
- 이미 공개된 정보
- 형식 샘플 (실제 데이터 대신 예시 데이터)

## 프롬프트 로그 주의

gigang-skills를 설치하면 Claude Code 세션의 프롬프트·응답이
`~/.claude/logs/prompts/` 에 자동으로 저장됩니다.

- 이 로그 파일은 평문(텍스트)으로 저장됩니다
- 프롬프트에 비밀번호나 API 키 등을 직접 입력하지 마세요
- 로그 파일이 다른 사람 손에 들어가면 입력한 내용이 노출될 수 있습니다
