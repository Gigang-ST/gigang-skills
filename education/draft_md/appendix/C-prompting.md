# 부록 C: 프롬프트 잘 쓰는 법

> AI에게 명확하게 지시하는 방법을 설명합니다.

## 왜 프롬프트가 중요한가?

같은 AI라도 지시를 어떻게 하느냐에 따라 결과가 완전히 달라진다.
AI 도구를 쓰는 실력 = 프롬프트 실력이라고 봐도 무방하다.

## 나쁜 프롬프트 vs 좋은 프롬프트

**나쁜 예:**
```
보고서 써줘
```
→ AI가 어떤 보고서인지, 어떤 형식인지 몰라서 엉뚱하게 씀

**좋은 예:**
```
3월 운동 데이터를 항목별로 비교한 1페이지 요약 보고서를 써줘.
형식: 제목, 항목별 실적 표, 총평 3줄.
독자: 팀 운영진.
```

## 좋은 프롬프트의 구성요소

### 1. 역할 지정
```
너는 10년 경력의 스포츠 트레이너야.
```

### 2. 맥락 제공
```
우리 팀은 러닝 기반 스포츠 팀이야.
이번 달 신규 멤버 3명이 합류했어.
```

### 3. 구체적 요청
```
신규 멤버들에게 보낼 환영 메시지 초안을 써줘.
```

### 4. 출력 형식 지정
```
형식: 제목 + 본문 3단락 + 서명.
길이: 200자 이내.
톤: 따뜻하고 친근하게.
```

## 업무별 프롬프트 예시

### 보고서 작성
```
[파일 내용 붙여넣기]
위 데이터를 바탕으로 팀 운영진 보고용 1페이지 요약을 만들어줘.
포함 항목: 핵심 지표 3개, 이슈사항, 다음 달 계획.
```

### 메일 초안
```
수신: [담당자명]
목적: 대회 참가 일정 조율
배경: 지난 모임에서 처음 만났고, 함께 참가 의향을 밝혔음
톤: 친근하지만 정중하게, 간결하게
```

### 데이터 정리
```
아래 엑셀 데이터에서:
1. 중복 항목 제거
2. 날짜 형식을 YYYY-MM-DD로 통일
3. 수치 열을 천 단위 구분기호 추가
결과를 새 파일로 저장해줘.
```

## 프롬프트 저장 팁

자주 쓰는 프롬프트는 텍스트 파일에 저장해두고 복사해서 쓰자.

```
C:\WORK\my-prompts\
  보고서-요약.txt
  메시지-초안.txt
  데이터-정리.txt
```
