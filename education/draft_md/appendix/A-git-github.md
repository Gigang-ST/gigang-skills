# 부록 A: Git & GitHub 입문

> 버전 관리가 무엇인지, 왜 설치했는지 설명합니다.

## Git이란?

**비유:** 문서의 역사 기록 시스템.
Word의 "변경 내용 추적"처럼, 파일의 모든 변경 이력을 저장한다.
언제든 이전 상태로 돌아갈 수 있다.

## GitHub란?

**비유:** Git 저장소를 위한 클라우드 드라이브.
내 컴퓨터의 Git 저장소를 인터넷에 올려서 팀과 공유한다.

## 왜 설치했는가?

gigang-skills는 GitHub에 있는 프로젝트다.
설치 과정에서 GitHub에서 파일을 내려받았기 때문에 필요했다.
Claude Code도 파일 관리에 Git을 활용한다.

## 기본 개념

| 용어 | 비유 | 설명 |
|------|------|------|
| Repository (저장소) | 프로젝트 폴더 | 파일 + 변경 이력 |
| Commit | 저장 | 현재 상태를 기록 |
| Clone | 다운로드 | 저장소를 내 PC에 복사 |
| Pull | 업데이트 | 최신 변경사항 받기 |
| Push | 업로드 | 내 변경사항 올리기 |

## 자주 쓰는 명령어

```powershell
# 저장소 내려받기
git clone [주소]

# 최신 업데이트 받기
git pull

# 변경사항 저장
git add .
git commit -m "변경 내용 설명"

# 변경사항 서버에 올리기
git push
```
