# 부록 G: 프로젝트 폴더·파일 관리

> 파일과 폴더를 어떻게 구성하면 Claude Code가 더 잘 도와주는지 설명합니다.

## 왜 폴더 구조가 중요한가?

Claude Code는 프로젝트 폴더를 통째로 읽는다.
폴더가 잘 정리되어 있으면 AI가 맥락을 더 잘 이해하고
더 정확한 결과를 낸다.

**비유:** 책상이 정리된 직원 vs 어수선한 직원에게 업무 지시할 때의 차이.

기강 스킬에는 `/gigang-folder-guide` 명령으로 AI가 폴더 구조 잡는 것을 도와줍니다.

## 흔히 쓰는 폴더명과 의미

| 폴더명 | 역할 | 예시 |
|--------|------|------|
| `docs/` | 문서, 설명자료 | 계획서, 회의록 |
| `refs/` | 참고자료, 외부 데이터 | 외부 자료, 비교 데이터 |
| `output/` | 결과물, 산출물 | 완성된 보고서, 분석 결과 |
| `data/` | 원본 데이터 | 엑셀 파일, CSV |
| `plans/` | 계획서, 기획 문서 | 프로젝트 계획, 일정표 |
| `templates/` | 양식, 템플릿 | 보고서 양식, 메시지 템플릿 |

## 프로젝트 예시

### 월간 보고서 프로젝트

```
C:\WORK\월간보고서-2026-05\
  data\
    활동실적.xlsx
    참여현황.xlsx
  refs\
    전월보고서.pdf
    목표수치.xlsx
  templates\
    보고서양식.docx
  output\
    (완성된 보고서가 여기 저장됨)
```

Claude Code 실행 후:
```
data 폴더의 엑셀 파일을 읽고
templates의 양식에 맞춰서
output에 5월 활동 보고서 초안 만들어줘.
refs의 전월 보고서도 참고해줘.
```

### AI 활용 프로젝트 관리

```
C:\WORK\ai-활용\
  prompts\
    보고서-요약.txt
    메시지-초안.txt
  output\
    결과물들\
  notes\
    메모.md
```

## Claude Code가 폴더를 활용하는 방식

Claude Code는 현재 폴더의 파일 목록을 자동으로 파악한다.
그래서 "data 폴더 읽어"라고 하면 알아서 찾는다.

프로젝트 시작 시 좋은 습관:
```
지금 이 폴더 구조를 파악하고,
어떤 작업을 도와줄 수 있는지 알려줘.
```

## 처음 시작할 때

새 프로젝트를 시작할 때 Claude Code에게:
```
[프로젝트 이름] 프로젝트를 시작하려고 해.
[목적 설명]
적절한 폴더 구조를 만들어줘.
```
→ AI가 프로젝트 성격에 맞는 폴더 구조를 제안해준다.

또는 기강 스킬 명령 사용:
```
/gigang-folder-guide
```
