# 기강 스킬 활용

> gigang-skills가 무엇인지, 팀 도구를 어떻게 활용하는지 설명합니다.

## gigang-skills란?

기강 러닝팀 멤버들이 함께 만들고 공유하는 Claude Code 확장 도구 모음.
`/gigang-init`으로 설치하면 아래 기능들이 자동으로 활성화됩니다.

| 명령 | 용도 |
|------|------|
| `/gigang-init` | 신규 멤버 환경 초기 셋업 |
| `/gigang-help` | 사용법 안내 |
| `/gigang-version` | 설치된 버전 및 업데이트 상태 확인 |
| `/gigang-folder-guide` | 대화형 폴더 구조 가이드 |
| `/gigang-report` | 버그·개선 아이디어를 GitHub Issue로 제출 |

## 자동 로그 기록

gigang-skills 설치 후 Claude Code를 쓰면 **자동으로 세션 기록이 저장**됩니다.
별도 명령이 필요 없습니다.

저장 위치: `~/.claude/logs/prompts/<프로젝트명>/YYYY-MM-DD.md`

이 로그 덕분에:
- 지난주에 뭘 했는지 언제든 조회 가능
- 팀 활동 통계 파악 가능
- 어떤 작업에 AI를 활용했는지 되돌아볼 수 있음

## 어제 뭐했더라?

Claude Code에서 자연어로 물어보면 된다:

```
어제 뭐했더라
```
```
지난주에 뭐했지
```
```
5/13에 뭐했더라
```

→ AI가 로그 파일을 읽고 작업 내역을 요약해줌

자세한 사용법: [부록 H. gigang-뭐했더라](appendix/H-gigang-뭐했더라.md)

## 자동 업데이트

Claude Code 세션을 시작할 때마다 gigang-skills가 자동으로 업데이트됩니다.
새 기능이 추가되면 별도 설치 없이 바로 사용 가능.

## 개선 제안 & 버그 제보

쓰다가 불편한 점이나 아이디어가 생기면:

```
/gigang-report
```

→ 대화형으로 내용을 입력하면 GitHub Issue가 자동으로 생성됩니다.
팀 전체에 공유되어 함께 개선할 수 있습니다.

## 실습 과제

Claude Code에서 아래를 해보세요:

**1.** 오늘 설치한 것을 시작으로 로그가 쌓이는지 확인:
```
어제 뭐했더라
```
(아직 로그가 없으면 "기록 없음"으로 나옵니다 — 정상)

**2.** 버전 확인:
```
/gigang-version
```

**3.** 폴더 구조 가이드 체험:
```
/gigang-folder-guide
```
